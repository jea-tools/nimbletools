# NimbleTools 隐私说明

最后更新：2026-05-21

NimbleTools 当前是一个本地运行的 macOS 桌面工具箱。这个说明描述当前开源仓库中的应用行为。

## 核心原则

- 应用不内置账号系统。
- 应用不内置遥测、统计或广告 SDK。
- 应用不会自动上传你的剪贴板、截图、文件内容或 curl 请求历史。
- 大部分工具在本机完成处理。

## 本地保存的数据

以下数据会保存在你的本机：

- 剪贴板历史：文本、文件路径、图片缓存路径和预览信息。
- 剪贴板图片缓存：从剪贴板读取到的图片会保存到应用缓存目录。
- Curl 工作台数据：项目、文件夹、请求、请求头、查询参数、请求体、响应头、响应体、错误信息和历史记录。
- 应用偏好：主题、语言、快捷键配置等。

这些数据通常位于系统分配给 NimbleTools 的应用数据目录和缓存目录中。具体路径由 Tauri 和操作系统决定。

## 网络访问

应用没有后台遥测或自动上报。

会发生网络访问的场景：

- 你在 `Curl 请求` 工具中主动发送请求。
- 你点击应用中或文档中的外部链接。
- 系统或开发环境为了依赖安装、构建、更新而访问网络，例如 `npm install` 或 Cargo 下载依赖。

`Curl 请求` 工具会按照你填写的 URL、方法、请求头、查询参数和请求体发送请求，并把历史记录保存在本机 SQLite 数据库中。

## 系统权限

根据你使用的功能，macOS 可能要求授权：

- 屏幕录制权限：用于截图。
- 辅助功能权限：用于快捷粘贴等自动化操作。
- 剪贴板访问：用于剪贴板历史。
- 文件访问：用于你主动选择的文件读写。

## 如何清理数据

你可以在应用内清理剪贴板历史或 curl 历史。也可以删除系统应用数据目录中的 NimbleTools 数据。

如果你不确定数据目录在哪里，可以从源码中搜索：

```bash
rg "app_data_dir|app_cache_dir" src-tauri/src
```

当前主要数据文件：

- `clipboard.db`
- `curl.db`
- `clipboard_cache/`

## 开源仓库隐私边界

公开仓库不应包含：

- 本地数据库。
- 构建缓存。
- 打包产物。
- 签名证书。
- 私钥。
- `.env` 文件。
- 个人路径或内部过程文档。

根目录 `.gitignore` 已经排除了这些常见内容。发布前仍建议运行一次敏感信息扫描。
